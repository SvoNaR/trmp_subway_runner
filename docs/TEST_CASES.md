# ТЕСТОВЫЕ КЕЙСЫ ДЛЯ МОБИЛЬНОГО ПРИЛОЖЕНИЯ "SUBWAY RUNNER"

## 1. Тестирование регистрации пользователя

### Тест-кейс TC-REG-001
**Название:** Успешная регистрация нового пользователя  
**Предусловие:** Пользователь не зарегистрирован в системе  
**Шаги:**
1. Открыть приложение
2. Нажать "Зарегистрироваться"
3. Ввести имя пользователя: "TestUser"
4. Ввести email: "test@example.com"
5. Ввести пароль: "password123"
6. Подтвердить пароль: "password123"
7. Нажать кнопку "Зарегистрироваться"

**Ожидаемый результат:** 
- Сообщение об успешной регистрации
- Переход на экран входа

---

### Тест-кейс TC-REG-002
**Название:** Регистрация с существующим email  
**Предусловие:** Пользователь с таким email уже существует  
**Шаги:**
1. Открыть приложение
2. Нажать "Зарегистрироваться"
3. Ввести данные существующего пользователя
4. Нажать кнопку "Зарегистрироваться"

**Ожаемый результат:** 
- Сообщение об ошибке "Username or email already exists"

---

### Тест-кейс TC-REG-003
**Название:** Регистрация с несовпадающими паролями  
**Шаги:**
1. Ввести пароль: "password123"
2. Ввести подтверждение: "password456"
3. Нажать кнопку "Зарегистрироваться"

**Ожидаемый результат:** 
- Сообщение "Пароли не совпадают"

---

**Предусловие:** Пользователь находится на главном экране

**Ожидаемый результат:**

- Отображение персонажа на стартовой позиции

---

### Тест-кейс TC-GAME-002
**Название:** Управление движением (свайп влево)  
**Предусловие:** Игра активна, персонаж в центральной полосе

1. Выполнить свайп влево

**Ожидаемый результат:**
- Персонаж перемещается на левую полосу

---

### Тест-кейс TC-GAME-003
**Название:** Управление движением (свайп вправо)  
**Предусловие:** Игра активна, персонаж в левой полосе  
**Шаги:**
1. Выполнить свайп вправо

**Ожидаемый результат:**
- Персонаж перемещается в центральную полосу

---

### Тест-кейс TC-GAME-004
**Название:** Сбор монет  
**Предусловие:** Игра активна, перед персонажем монета  
**Шаги:**
1. Переместиться на полосу с монетой
2. Достичь монеты

**Ожидаемый результат:**
- Монета исчезает
- Счетчик монет увеличивается на 1
- Общий счет увеличивается на 10

---

### Тест-кейс TC-GAME-005
**Название:** Столкновение с препятствием  
**Предусловие:** Игра активна, перед персонажем препятствие  
**Шаги:**
1. Не менять полосу движения
2. Достичь препятствия

**Ожидаемый результат:**
- Игра останавливается
- Отображается экран "ИГРА ОКОНЧЕНА"
- Показываются финальные результаты

## 5. Тестирование API

### Тест-кейс TC-API-001
**Название:** POST /api/register - успешная регистрация  
**Метод:** POST  
**Endpoint:** /api/register  
**Тело запроса:**
```json
{
  "username": "TestUser",
  "email": "test@example.com",
  "password": "password123"
}
```

**Ожидаемый результат:**
- Статус 201
- Ответ: `{"message": "Registration successful", "userId": 1}`

---

### Тест-кейс TC-API-002
**Название:** POST /api/login - успешный вход  
**Метод:** POST  
**Endpoint:** /api/login  
**Тело запроса:**
```json
{
  "email": "test@example.com",
  "password": "password123"
}
```

**Ожидаемый результат:**
- Статус 200
- Ответ содержит токен и данные пользователя

---

### Тест-кейс TC-API-003
**Название:** POST /api/scores - сохранение результата  
**Метод:** POST  
**Endpoint:** /api/scores  
**Заголовки:** Authorization: Bearer {token}  
**Тело запроса:**
```json
{
  "score": 1500,
  "coins": 25,
  "distance": 500
}
```

**Ожидаемый результат:**
- Статус 201
- Результат сохранен в базе данных

---

### Тест-кейс TC-API-004
**Название:** GET /api/leaderboard - получение лидеров  
**Метод:** GET  
**Endpoint:** /api/leaderboard  

**Ожидаемый результат:**
- Статус 200
- Массив из топ-10 игроков
